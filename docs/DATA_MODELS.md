## Organization

The core domain entity. Represents a company profile created and managed by a user.

```ts
interface Organization {
  id: string;                    // UUID
  name: string;                  // Display name, e.g. "Acme Corp"
  domain: string;                // Web domain, e.g. "acme.com"
  industry: string;              // Industry label, e.g. "Managed Services"
  size_range: string;            // Headcount band, e.g. "51-200"
  regions: string[];             // Operating regions, e.g. ["North America", "EMEA"]

  brand: {
    colors: {
      primary: string;           // Hex color, e.g. "#1E40AF"
      secondary: string;
      accent: string;
    };
    font_vibe: string;           // Personality descriptor, e.g. "Modern Sans-serif"
    logo_url?: string;           // Optional URL to logo image
  };

  voice_profile: {
    tone: 'Professional' | 'Friendly' | 'Technical' | 'Urgent';
    reading_level: 'Grade 8' | 'High School' | 'PhD';
    banned_phrases: string[];    // Words/phrases the AI must never use
  };

  services: ServiceItem[];
  team: TeamMember[];
  compliance: ComplianceItem[];
  proof: ProofItem[];

  created_at: string;            // ISO 8601 timestamp
}
```

### Field Notes

| Field | Notes |
|---|---|
| `id` | Generated by Supabase (`uuid_generate_v4()`) or a local `crypto.randomUUID()` in demo mode |
| `brand.font_vibe` | Free-text descriptor. Not a CSS font-family — used to inform AI tone only |
| `voice_profile.banned_phrases` | Injected directly into exported prompts as a hard prohibition |
| `proof` | Metrics used to demonstrate capability (e.g. "99.9% uptime") |

## OrganizationVersion

A snapshot of an `Organization` at a point in time, used by the version-history feature.

```ts
interface OrganizationVersion {
  id: string;                        // UUID
  organization_id: string;           // References Organization.id
  version_number: number;            // Monotonically increasing (1, 2, 3 …)
  data: Partial<Organization>;       // Snapshot of changed fields
  created_by: string;                // User ID of the author
  created_at: string;                // ISO 8601 timestamp
  change_summary: string;            // Human-readable description of changes
}
```

### Usage

The OrgDetail page stores each save as a new `OrganizationVersion`. Users can compare any two versions and revert the organization to an earlier snapshot.

## Persona

An AI agent configuration that is attached to an `Organization`. A persona defines the AI's role, platform target, behavioral constraints, and few-shot examples.

```ts
interface Persona {
  id: string;                        // UUID
  organization_id: string;           // References Organization.id

  name: string;                      // Display name, e.g. "L1 Support Agent"
  platform: 'Claude' | 'Google' | 'Microsoft' | 'Custom';
  type: 'Internal' | 'External';
  role_archetype: 'Sales' | 'Support' | 'IT' | 'Security' |
                  'HR' | 'Ops' | 'Finance' | 'Engineering';
  mode: 'Front-of-House' | 'Back-of-House';

  constraints: string[];             // Things the AI must NOT do
  capabilities: string[];            // Things the AI IS authorised to do

  examples: {
    input: string;                   // Sample user message
    output: string;                  // Ideal AI response
    type: 'Standard' | 'Edge Case';
  }[];

  updated_at: string;                // ISO 8601 timestamp
}
```

### Field Notes

| Field | Notes |
|---|---|
| `platform` | Determines which adapter function is used at export time |
| `type` | `Internal` means the audience is employees; `External` means customers |
| `mode` | `Front-of-House` = directly interacts with end users; `Back-of-House` = runs behind the scenes |
| `constraints` | Injected into the export as a strict prohibition list |
| `capabilities` | Injected into the export as an explicit authorisation list |
| `examples` | Directly translated into few-shot examples in the exported prompt |

## Supporting Types

### ServiceItem

A named service offering that the organization provides.

```ts
interface ServiceItem {
  id: string;
  name: string;            // e.g. "Cloud Infrastructure Management"
  description: string;     // One-paragraph overview
  outcomes: string[];      // Measurable results, e.g. ["99.9% uptime SLA"]
}
```

### TeamMember

A role within the organization's team structure.

```ts
interface TeamMember {
  id: string;
  role: string;            // e.g. "Solutions Architect"
  count: number;           // Number of people in this role
  description: string;     // Responsibilities summary
}
```

### ComplianceItem

A compliance framework or certification tracked by the organization.

```ts
interface ComplianceItem {
  id: string;
  framework: string;       // e.g. "ISO 27001", "SOC 2 Type II", "HIPAA"
  status: 'Certified' | 'Compliant' | 'In Progress';
  details: string;         // Notes on scope, audit date, etc.
}
```

### ProofItem

A verifiable metric that demonstrates organizational capability.

```ts
interface ProofItem {
  id: string;
  metric: string;          // e.g. "System Uptime"
  value: string;           // e.g. "99.97%"
  context: string;         // Qualifying statement, e.g. "measured over 24 months"
  verified: boolean;       // Whether the metric has been independently verified
}
```

### Collaborator

Represents another user currently viewing or editing the same organization. Used to power real-time presence indicators.

```ts
interface Collaborator {
  id: string;
  name: string;
  avatar_url?: string;
  color: string;           // Hex color used for the presence dot/avatar ring
  status: 'active' | 'idle';
  current_path: string;    // Current route the collaborator is viewing
}
```

### ExportResult

The output of an adapter service call.

```ts
interface ExportResult {
  format: string;    // e.g. "Claude XML", "Google Markdown"
  content: string;   // The full prompt string ready to copy or download
}
```

### WizardStep

A union type representing the valid steps in the organization-creation wizard.

```ts
type WizardStep =
  | 'identity'
  | 'brand'
  | 'services'
  | 'team'
  | 'compliance'
  | 'voice'
  | 'review';
```

The wizard progresses through these steps in order. The `review` step is a final summary before saving.

## Type Relationships

```
Organization
├── brand { colors, font_vibe, logo_url }
├── voice_profile { tone, reading_level, banned_phrases }
├── services[]      → ServiceItem
├── team[]          → TeamMember
├── compliance[]    → ComplianceItem
└── proof[]         → ProofItem

Persona
└── organization_id → Organization.id

OrganizationVersion
└── organization_id → Organization.id
    └── data: Partial<Organization>
```

An `Organization` is the central entity. All other entities reference it by `organization_id`. `OrganizationVersion` stores partial snapshots, allowing fields to be selectively versioned.
